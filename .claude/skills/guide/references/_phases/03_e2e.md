# フェーズ詳細: E2Eテスト

SKILL.md の「フロー D」の詳細ワークフロー。
テストフレームワーク: Playwright
テストディレクトリ: packages/web/e2e/

## E2Eメニュー表示形式

```
E2Eテスト — 何をしますか?

  1. 追加   — 新しいテストを作成
  2. 変更   — 既存テストを修正
  3. 実行   — テストを実行する
```

## Phase 追加（D-Add）

### D-Add-2: リファレンスの差異確認
references/e2e/index.md を読み込み、実際の packages/web/e2e/ と比較。
差異があれば:
```
index.mdと現在のファイルに差異があります:
  追加されているファイル: xxx.spec.ts
  削除されているファイル: yyy.spec.ts

index.mdを更新しますか?
  1. 今すぐ更新する
  2. 後で更新する（このまま進む）
```

### D-Add-3: テストシナリオ案
```
テストシナリオ案:

  シナリオ1: {操作} → {期待結果}
  シナリオ2: {操作} → {期待結果}
  （エラーケースも含める）

このシナリオでOKですか?
```

### D-Add-5: 実行確認
実装後に「実行しますか?」を確認。
→ Yes: D-Run-2へ（追加したテストのみ実行）

## Phase 変更（D-Mod）

### D-Mod-1: 変更対象の選択
references/e2e/index.md の一覧を表示して選択させる。

## Phase 実行（D-Run）

### D-Run-1: テスト選択
references/e2e/index.md の一覧を表示:
```
実行するテストを選択:
  1. 01-parent-registration.spec.ts
  2. 02-child-invitation.spec.ts
  ...
  A. すべて実行
```

### D-Run-2: 実行コマンド
```bash
cd packages/web && npx playwright test {ファイル名}
# すべて実行の場合
cd packages/web && npx playwright test
```

### D-Run-3: 結果表示
テスト結果をそのまま表示し、失敗があれば原因を簡潔に説明。
