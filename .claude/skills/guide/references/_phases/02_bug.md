# フェーズ詳細: バグ修正

SKILL.md の「フロー B」の詳細ワークフロー。

## バグ修正フロー

### B-1: バグ内容の入力
ユーザーに以下を自由記述で入力してもらう:
- どんな操作をしたか
- 期待した動作
- 実際の動作（エラーメッセージがあれば含める）

### B-2: 関連リファレンスの読み込み
サブエージェントに以下を渡して推定させる:
- バグの内容テキスト
- 推定される関連機能・レイヤー（画面/API/DB）

結果のみメインエージェントに返す。

### B-3: 原因調査
- 関連するコードファイルを読み込む
- エラーログ・スタックトレースがあれば分析
- アーキテクチャの5層（client/route/service/query/db）のどこで発生しているか特定

### B-4: 原因候補の質問シート

提示形式:
```
原因候補:
  1. [原因A] — {ファイルパス:行番号}
  2. [原因B] — {ファイルパス:行番号}
  3. その他の可能性

確認させてください:
  Q1. {追加で知りたいこと}
  Q2. {追加で知りたいこと}
```

ユーザーの回答で絞り込めなければ再質問（回数制限なし）。
質問内容は `~/.claude/skill-memory/guide/sessions/{YYYYMMDDHHMMSS}_bug_{機能名}.md` に保存。

### B-5: 修正方針
原因確定後:
- 修正箇所とアプローチを提示
- 副作用・影響範囲を明示
- ユーザーの承認を得てから実装へ

### B-6: 実装

### B-7: E2E確認
「E2Eテストを実行しますか?」
→ Yes: どのテストを実行するか選択 → 実行
→ No: スキップ
